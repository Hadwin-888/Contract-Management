## 1. 系统概述

### 1.1 系统简介

AI 合同管理平台是一套现代化的企业合同全生命周期管理系统，集成 AI 智能审核能力，帮助企业管理合同的创建、存储、审核、提醒和统计分析等全流程工作。

### 1.2 核心功能

| 功能模块 | 说明 |
|---------|------|
| 合同管理 | 合同的新建、编辑、删除、搜索、筛选、导出 |
| AI 审核 | 上传合同文件，AI 自动提取信息并分析风险 |
| 合同提醒 | 即将到期的合同自动提醒 |
| 统计分析 | 合同数据的可视化统计与趋势分析 |
| 系统设置 | 用户管理、部门管理、AI 模型配置、审核模板管理 |

### 1.3 技术架构

- **前端：** Vue 3 + TypeScript + Element Plus + TailwindCSS
- **后端：** Node.js + Express + SQLite
- **AI 引擎：** 支持 DeepSeek / Minimax / 通义千问
- **部署方式：** 支持 Docker 容器化部署

## 2. 登录与账号

### 2.1 登录页面

打开系统后，进入登录页面。页面包含：

- **卡通助手：** 页面左侧的卡通角色会随您的操作做出互动反应
- **用户名输入框：** 输入您的账号用户名
- **密码输入框：** 输入您的密码
- **登录按钮：** 点击登录系统

**快捷操作：** 输入完成后按 Enter 键可直接登录。

### 2.2 登录交互

卡通角色会根据您的操作做出不同反应：

| 您的操作 | 角色反应 |
|---------|---------|

点击页面右下角的 **图片按钮** 🖼️，可：

1. **选择本地图片** — 上传自己喜欢的图片作为登录背景
2. **恢复默认渐变** — 恢复到默认的动态渐变背景

> 自定义背景会自动保存，下次打开登录页时自动加载。

### 2.4 默认管理员账号

| 用户名 | 密码 | 角色 |
|-------|------|------|
| `admin` | `admin123` | 系统管理员 |

## 3. 角色与权限

### 3.1 角色体系

系统包含 4 级角色，权限从低到高：

| 角色 | 英文标识 | 权限范围 |
|------|---------|---------|
| 部门文员 | `clerk` | 仅可访问 AI 审核模块 |
| 部门负责人 | `head` | 审核 + 仪表盘 + 合同提醒（仅本部门数据） |
| 合同管理员 | `admin` | 合同管理 + 审核 + 统计 + 审核模板配置 |
| 系统管理员 | `super_admin` | 全部权限，包括用户管理、系统设置 |

### 3.2 数据隔离规则

- **部门文员** 和 **部门负责人** 只能查看和操作 **本部门** 的合同数据
- **合同管理员** 和 **系统管理员** 可以查看 **全部部门** 的数据

### 3.3 菜单可见性

不同角色看到的左侧菜单项不同：

| 菜单 | clerk | head | admin | super_admin |
|------|-------|------|-------|-------------|
| 首页 | ❌ | ✅ | ✅ | ✅ |
| 合同管理 | ❌ | ❌ | ✅ | ✅ |
| AI 审核 | ✅ | ✅ | ✅ | ✅ |
| 合同提醒 | ❌ | ✅ | ✅ | ✅ |
| 统计分析 | ❌ | ❌ | ✅ | ✅ |
| 系统设置 | ❌ | ❌ | ❌ | ✅ |

## 5. 合同管理

### 5.1 合同列表

合同管理页面以表格形式展示所有合同，支持多维度筛选和分页浏览。

**表格列说明：**

| 列 | 说明 |
|----|------|
| 合同编号 | 合同唯一编号 |
| 合同名称 | 合同全称 |
| 甲方 | 甲方（采购方/委托方）名称 |
| 乙方 | 乙方（供应方/受托方）名称 |
| 金额 | 合同金额（万元显示） |
| 不含税金额 | 不含税合同金额 |
| 税率 | 增值税税率 |
| 质保金 | 质保金比例或金额 |
| 起始日期 | 合同开始日期 |
| 结束日期 | 合同结束日期 |
| 合同期限 | 合同有效期描述 |
| 保险情况 | 保险信息 |
| 保险日期 | 保险到期日期 |
| 跟进部门 | 负责跟进的部门 |
| 费用部门 | 承担费用的部门 |
| 费用代码 | 费用科目代码 |
| 状态 | 进行中 / 草稿 / 已过期 / 已终止 |
| 风险 | 低风险 / 中风险 / 高风险 |
| 操作 | 编辑、删除、上传文件等 |

### 5.2 筛选功能

页面顶部提供多种筛选条件：

- **搜索框** — 按合同名称或合同编号搜索
- **合同状态** — 按状态筛选（进行中/草稿/已过期/已终止）
- **风险等级** — 按风险等级筛选（低/中/高）
- **跟进部门** — 按跟进部门筛选
- **费用部门** — 按费用部门筛选
- **金额范围** — 按合同金额区间筛选

> 部门文员和部门负责人只能看到本部门的合同，跟进部门筛选自动锁定。

### 5.3 新建合同

点击页面右上角的 **「新建合同」** 按钮，弹出合同编辑对话框，填写以下信息：

**基本信息：**
- 合同编号（必填）
- 合同名称（必填）
- 甲方名称（必填）
- 乙方名称（必填）
- 合同类型（必填）：采购 / 服务 / 租赁 / 营销 / 技术 / 咨询 / 人力资源 / 物流
- 合同状态：进行中 / 草稿 / 已过期 / 已终止

**金额信息：**
- 合同金额（必填）
- 不含税金额
- 税率(%)
- 质保金情况

**时间信息：**
- 起始日期（必填）
- 结束日期（必填）
- 合同期限

**保险信息：**
- 保险情况
- 保险日期

**部门信息：**
- 跟进部门（必填）
- 费用部门（必填）
- 费用代码（必填）

**风险等级：**
- 低风险 / 中风险 / 高风险

### 5.4 AI 智能提取

在新建合同时，点击 **「AI 提取合同内容」** 按钮：

1. 选择要上传的合同文件（支持 PDF、DOCX、TXT、JPG、PNG）
2. AI 自动识别并填写：合同名称、甲乙双方、金额、日期等信息
3. 确认提取结果后保存

> AI 提取需要配置有效的 API Key，详见 [9.2 AI 模型配置](#92-ai-模型配置)。

### 5.5 编辑合同

在合同列表的操作列点击 **「编辑」** 按钮，可修改合同的全部字段。

### 5.6 删除合同

在合同列表的操作列点击 **删除图标** 🗑️，确认后删除合同。

> ⚠️ 删除操作不可撤销。

### 5.7 文件上传

每个合同支持上传两类文件：

| 文件类型 | 说明 | 操作 |
|---------|------|------|
| 合同扫描件 | 合同正本扫描件（PDF/DOCX） | 上传 / 查看 / 删除 |
| 保单扫描件 | 保险单扫描件 | 上传 / 查看 / 删除 |

支持的文件格式：PDF、DOCX、DOC、TXT、JPG、PNG，单文件最大 20MB。

### 5.8 导出 CSV

点击页面右上角的 **「导出 CSV」** 按钮：

- 导出当前筛选条件下的全部合同数据
- 文件格式为 CSV（UTF-8 编码，带 BOM，Excel 可直接打开）
- 包含 20 个字段的完整合同信息
- 文件名自动包含导出日期

## 6. AI 审核

### 6.1 审核流程

AI 审核的整体流程：

```
上传合同文件 → AI 提取合同信息 → AI 分析风险 → 生成审核报告
```

### 6.2 上传并审核

点击 **「上传并审核」** 按钮：

1. **选择文件** — 拖拽或点击选择合同文件（PDF、DOCX、TXT）
2. **选择文件类型** — 选择对应的合同类型（采购/服务/租赁等）
3. **提交审核** — 系统自动完成以下步骤：
   - 创建审核草稿合同
   - 上传文件并关联
   - AI 提取合同信息（名称、双方、金额、日期等）
   - AI 根据审核规则进行风险分析
   - 生成审核报告和文件概况

### 6.3 审核已有合同

点击 **「审核已有合同」** 按钮，系统会自动查找尚未审核的合同并触发 AI 分析。

### 6.4 审核结果

每条审核记录包含：

| 项目 | 说明 |
|------|------|
| 风险评分 | 0-100 分，越高越安全 |
| 发现问题 | AI 识别出的问题数量 |
| 审核状态 | 通过（≥70分）/ 警告（50-69分）/ 未通过（<50分）|
| 分析报告 | AI 生成的详细分析报告（中文） |
| 改进建议 | 3-5 条具体的改进建议 |

### 6.5 查看审核详情

在审核列表中点击 **「查看详情」**，可查看完整的审核报告，包括：

- 合同基本信息
- 风险评分和状态
- AI 分析结果全文
- 改进建议列表

### 6.6 下载审核报告

支持下载两种类型的报告：

| 报告类型 | 格式 | 内容 |
|---------|------|------|
| 审核报告 | .txt | 风险评分、分析报告、改进建议 |
| 文件概况 | .txt | AI 生成的合同文件内容摘要 |

## 9. 系统设置

### 9.1 用户管理

**权限：** 仅系统管理员

管理所有系统用户，支持：

- **查看用户列表** — 展示用户名、姓名、邮箱、部门、角色
- **新建用户** — 创建新用户账号
- **编辑用户** — 修改用户信息和角色
- **删除用户** — 移除用户账号
- **重置密码** — 重置用户登录密码

**用户字段说明：**

| 字段 | 说明 |
|------|------|
| 用户名 | 登录使用的账号名 |
| 姓名 | 用户真实姓名 |
| 邮箱 | 用户电子邮箱 |
| 部门 | 所属部门 |
| 角色 | 系统角色（文员/负责人/管理员/超级管理员） |

### 9.2 AI 模型配置

**权限：** 系统管理员

配置 AI 审核使用的模型和 API Key：

| 配置项 | 说明 |
|-------|------|
| 模型选择 | DeepSeek / Minimax / 通义千问 |
| API Key | 对应模型的 API 密钥 |

> API Key 在数据库中加密存储，非系统管理员查看时自动脱敏。

### 9.3 审核模板管理

**权限：** 合同管理员、系统管理员

为各类型合同配置 AI 审核规则和文件概况规则（Markdown 格式）。

**支持的类型：**

| 合同类型 | 说明 |
|---------|------|
| 采购 | 采购合同审核规则 |
| 服务 | 服务合同审核规则 |
| 租赁 | 租赁合同审核规则 |
| 营销 | 营销合同审核规则 |
| 技术 | 技术合同审核规则 |
| 咨询 | 咨询合同审核规则 |
| 人力资源 | 人力资源合同审核规则 |
| 物流 | 物流合同审核规则 |

**操作说明：**

- **新建规则** — 点击「新增类型」创建新的合同类型并配置规则
- **编辑规则** — 修改已有审核规则的内容
- **预览规则** — 查看 Markdown 渲染效果
- **删除规则** — 完全删除该类型的审核配置

**审核规则使用 Markdown 格式，支持：**

```markdown
# 一级标题
## 二级标题
### 三级标题
- 列表项
1. 编号列表
```

**每个模板包含两部分：**

1. **审核规则** — AI 用于风险分析和评分的检查标准
2. **文件概况规则** — AI 用于生成合同文件概况的提取规则

> 更新规则后，新的 AI 审核将自动使用最新版本。

### 9.4 部门管理

**权限：** 系统管理员

管理企业组织架构：

- **查看部门列表** — 展示部门编码、简称、全称、负责人
- **新建部门** — 添加新的部门
- **编辑部门** — 修改部门信息
- **删除部门** — 移除部门

**部门字段：**

| 字段 | 说明 |
|------|------|
| 部门编码 | 部门唯一编码 |
| 简称 | 部门简称 |
| 全称 | 部门完整名称 |
| 负责人 | 部门负责人姓名 |

### 9.5 文件存储配置

**权限：** 系统管理员

配置文件存储相关设置：

| 配置项 | 说明 |
|-------|------|
| 合同文件路径 | 合同扫描件的存储目录 |
| 保单文件路径 | 保单扫描件的存储目录 |
| 文件命名规则 | 支持模板变量自动命名 |

**命名规则模板变量：**

| 变量 | 说明 |
|------|------|
| `{contractNo}` | 合同编号 |
| `{name}` | 合同名称 |
| `{partyA}` | 甲方名称 |
| `{partyB}` | 乙方名称 |
| `{contractId}` | 合同 ID |

默认命名规则：`{contractNo}{name}{partyB}`

## 10. 常见问题

### 10.1 登录失败

**问题：** 登录时提示"用户名或密码错误"  
**解决：** 
- 检查用户名和密码是否正确
- 联系系统管理员重置密码

### 10.2 AI 审核失败

**问题：** AI 审核时提示"API Key 未配置"  
**解决：**
1. 使用系统管理员账号登录
2. 进入「系统设置 → AI 模型配置」
3. 配置有效的 API Key 并保存

### 10.3 文件上传失败

**问题：** 上传文件时提示"不支持的文件类型"  
**解决：** 系统支持以下文件格式：
- PDF（.pdf）
- Word 文档（.doc, .docx）
- 纯文本（.txt）
- 图片（.jpg, .jpeg, .png）
- 单文件最大 20MB

### 10.4 合同数据看不到

**问题：** 登录后看不到某些合同数据  
**解决：** 
- 部门文员和部门负责人只能看到本部门的合同
- 如需查看其他部门数据，请联系管理员提升权限

### 10.5 导出 CSV 乱码

**问题：** 导出的 CSV 文件在 Excel 中打开乱码  
**解决：** 
- 文件使用 UTF-8 编码并包含 BOM 标记
- Excel 应自动识别，如仍有问题，请使用 Excel 的"数据 → 从文本/CSV 导入"功能

### 10.6 忘记密码

联系系统管理员重置密码。
